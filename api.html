API¶

The following is the API documentation.

recLeague.create_app() → Flask¶

Creates Flask application object.

This method sets the app configuration and imports app the blueprints.

Returns:

application object

Return type:

Flask

Routes¶

Below are the API routes. Flask template routes for the application are not listed .

GET /auth¶

Indicates whether the user is currently logged in.

Method is used by NGINX for authorized static routes.

Status Codes:

  • 200 OK – User is logged in.

  • 401 Unauthorized – User is not logged in.

GET /team-players/(team_id)¶

API route for getting players on team.

Args:

team_id (int): ID of team

Returns:

json: Array of user IDs on team.

Response Headers:

  • Content-Type – application/json

Status Codes:

  • 200 OK – Team has players.

  • 204 No Content – Team does not exist.

  • 400 Bad Request – Not valid team_id.

GET /teams-latest-game-score¶

API route gets latest game score between two teams.

Returns:

json: Object with fields team_1 and team_2 set to the score.

Query Parameters:

  • team-1 (int) – ID of team 1.

  • team-2 (int) – ID of team 2.

Status Codes:

  • 200 OK – Teams have a game between each other.

  • 204 No Content – No game between the teams was found.

  • 400 Bad Request – Team query parameters not valid.

Environment Variables¶

Flask Config Path¶

FLASK_CONFIG_PATH

Path to a configuration file used to set Flask app.config. Path should be absolute. See flask.Config.from_envvar for details about the config file.

League Config Path¶

LEAGUE_CONFIG_PATH

Path to a JSON configuration for league settings. See config for details about the JSON file. Supports either relative or absolute paths. Relative paths are from the directory flask-rec-league/config.

recLeague.config.SCORECARD_PICS_STATIC_PATH: str¶

Path to scorecard pics relative to the static folder.

This is dynamically created from SQLALCHEMY_DATABASE_URI by using the basename of the URI in combination with the full URI hashed to a 10 character hash. The two names are combined with a “_” as shown below.

{uri_basename}_{hashed_uri}

This naming scheme is beneficial when using multiple database URIs in order to avoid overwriting other databases images. The basename is to make the folder easily found by a human, and the hash is to make sure the path is unique for URI. Without the hash a collision could occur for example:

site:///test.db
- vs. -
site:///folder/test.db

recLeague.games.utils.is_game_user_modifiable(game: Game) → bool¶

Returns if the game is able to be modified by the current user.

Can only modify the game if current is user is an admin or if the current user is one of the players is in the game and the game hasn’t been verified yet.

Parameters:

game (Game) – Game to check if it is editable.

Returns:

True if game is able to edit by the current user, otherwise False

Return type:

bool
